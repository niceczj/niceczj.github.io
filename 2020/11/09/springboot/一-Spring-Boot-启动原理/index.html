(一) Spring Boot 启动原理

发表于 2020-11-09 分类于 主流框架 ， Spring ， Spring Boot Valine：

本文 Spring Boot 系列第一篇，主要从源码角度分析了 Spring Boot 启动的过程。

启动Jar包

Spring Boot 经过spring-boot-maven-plugin插件打包后，通过 java -jar便能开启一个 Web 服务，这是什么原理？
我们打包一个 Spring Boot Jar 包后查看下 Jar 包内的目录结构：

➜  target jar tf eureka-1.0-SNAPSHOT.jar
META-INF/
META-INF/MANIFEST.MF
......

接着查看MANIFEST.MF中的内容：

➜  target jar xf eureka-1.0-SNAPSHOT.jar META-INF/MANIFEST.MF
➜  target cat META-INF/MANIFEST.MF
Manifest-Version: 1.0
Spring-Boot-Classpath-Index: BOOT-INF/classpath.idx
Implementation-Title: eureka
Implementation-Version: 1.0-SNAPSHOT
Start-Class: com.example.eureka.EurekaServerApplication
Spring-Boot-Classes: BOOT-INF/classes/
Spring-Boot-Lib: BOOT-INF/lib/
Build-Jdk-Spec: 1.8
Spring-Boot-Version: 2.3.3.RELEASE
Created-By: Maven Jar Plugin 3.2.0
Main-Class: org.springframework.boot.loader.JarLauncher

Start-Class 是我们项目的 Application 类， Main-Class 则是启动入口类。
看下 JarLauncher 里面的 main 函数：

public static void main(String[] args) throws Exception {
  new JarLauncher().launch(args);
}

/**
 * ----------------------------------------
 * org.springframework.boot.loader.Launcher
 */
protected void launch(String[] args) throws Exception {
  // 创建一个指定归档文件的类加载器 LaunchedURLClassLoader
  // 加载 /BOOT-INF/lib 及 /BOOT-INF/lib 目录下的类
  ClassLoader classLoader = createClassLoader(getClassPathArchivesIterator());
  launch(args, launchClass, classLoader);
}

protected void launch(String[] args, String launchClass, ClassLoader classLoader) throws Exception {
  // 将 LaunchedURLClassLoader 设置成当前线程的类加载器
  Thread.currentThread().setContextClassLoader(classLoader);
  // 创建 MainMethodRunner 用于启动这个应用
  createMainMethodRunner(launchClass, args, classLoader).run();
}

/**
 * ------------------------------------------------
 * org.springframework.boot.loader.MainMethodRunner
 */
public void run() throws Exception {
  Class<?> mainClass = Class.forName(this.mainClassName, false, Thread.currentThread().getContextClassLoader());
  Method mainMethod = mainClass.getDeclaredMethod("main", String[].class);
  mainMethod.setAccessible(true);
  mainMethod.invoke(null, new Object[] { this.args });
}

Manifest.MF 提供了整个工程启动的元数据，配置的Main-Class启动类创建类加载器加载指定目录下的类，最后通过反射Start-Class的 main 方法，间接启动服务。

ApplicationContextInitializer

ApplicationContextInitializer 是执行 refresh 前的一个扩展点，会在SpringApplication#prepareContext执行过程中触发，通常可以做一些额外的初始化工作，如dubbo-spring-boot中会注册一个 BeanFactoryPostProcessor。

  1. DelegatingApplicationContextInitializer：获取配置文件的context.initializer.classes，并执行ApplicationContextInitializerinitializer。注意DelegatingApplicationContextInitializer的order是0，最先执行。

  2. SharedMetadataReaderFactoryContextInitializer：注册CachingMetadataReaderFactoryPostProcessor。
    CachingMetadataReaderFactoryPostProcessor 除了注册 SharedMetadataReaderFactoryBean 之外还为ConfigurationClassPostProcessor配置了metadataReaderFactory=internalCachingMetadataReaderFactory。

    private static class CachingMetadataReaderFactoryPostProcessor
  implements BeanDefinitionRegistryPostProcessor, PriorityOrdered {

  @Override
  public void postProcessBeanDefinitionRegistry(BeanDefinitionRegistry registry) throws BeansException {
    register(registry);
    configureConfigurationClassPostProcessor(registry);
  }

  private void register(BeanDefinitionRegistry registry) {
    // 注册 SharedMetadataReaderFactoryBean
    BeanDefinition definition = BeanDefinitionBuilder
      .genericBeanDefinition(SharedMetadataReaderFactoryBean.class, SharedMetadataReaderFactoryBean::new)
      .getBeanDefinition();
    registry.registerBeanDefinition(BEAN_NAME, definition);
  }

  private void configureConfigurationClassPostProcessor(BeanDefinitionRegistry registry) {
    try {
      BeanDefinition definition = registry
     .getBeanDefinition(AnnotationConfigUtils.CONFIGURATION_ANNOTATION_PROCESSOR_BEAN_NAME);
      definition.getPropertyValues().add("metadataReaderFactory", new RuntimeBeanReference(BEAN_NAME));
    }
  }
}

ApplicationListener

ApplicationListener 是广泛运用在 Spring Framework 中的事件驱动机制 ，可以1个事件广播多个接受者，属于设计模式中观察者模式。
ApplicationEvent定义事件，ApplicationEventPublisher发布事件，ApplicationListener接受事件。
还有很多特性比如顺序、注解、事务、异步等。读者可以参考(四) Spring Event。

private <T> Collection<T> getSpringFactoriesInstances(Class<T> type, Class<?>[] parameterTypes, Object... args) {
  ClassLoader classLoader = getClassLoader();
  // SPI
  Set<String> names = new LinkedHashSet<>(SpringFactoriesLoader.loadFactoryNames(type, classLoader));
  List<T> instances = createSpringFactoriesInstances(type, parameterTypes, classLoader, args, names);
  AnnotationAwareOrderComparator.sort(instances);
  return instances;
}

// 通过反射创建实例
private <T> List<T> createSpringFactoriesInstances(Class<T> type, Class<?>[] parameterTypes,
                                                   ClassLoader classLoader, Object[] args, Set<String> names) {
  List<T> instances = new ArrayList<>(names.size());
  for (String name : names) {
    try {
      Class<?> instanceClass = ClassUtils.forName(name, classLoader);
      Assert.isAssignable(type, instanceClass);
      Constructor<?> constructor = instanceClass.getDeclaredConstructor(parameterTypes);
      T instance = (T) BeanUtils.instantiateClass(constructor, args);
      instances.add(instance);
    }
    catch (Throwable ex) {
      throw new IllegalArgumentException("Cannot instantiate " + type + " : " + name, ex);
    }
  }
  return instances;
}

EventPublishingRunListener

SpringApplicationRunListener是内置的唯一实现。
与ApplicationListener不同，EventPublishingRunListener的身份更像是一个事件广播器的代理而不是一个接收者。

public class EventPublishingRunListener implements SpringApplicationRunListener, Ordered {
   // 通过 SimpleApplicationEventMulticaster广播
   private final SimpleApplicationEventMulticaster initialMulticaster;

   // 获得所有的 ApplicationListener 转发给 SimpleApplicationEventMulticaster 广播
   public EventPublishingRunListener(SpringApplication application, String[] args) {
		this.application = application;
		this.args = args;
        // 这里的SimpleApplicationEventMulticaster是没有taskExecutor的，只能串行执行
		this.initialMulticaster = new SimpleApplicationEventMulticaster();
		for (ApplicationListener<?> listener : application.getListeners()) {
			this.initialMulticaster.addApplicationListener(listener);
		}
	}
	@Override
	public void starting() {
		this.initialMulticaster.multicastEvent(new ApplicationStartingEvent(this.application, this.args));
	}

  @Override
	public void started(ConfigurableApplicationContext context) {
        // 当引用上下文创建完毕后，事件就归还给上下文来管理了
        // 这个Context中的事件广播器可以手动创建一个，也就是说可以添加线程池异步处理了
		context.publishEvent(new ApplicationStartedEvent(this.application, this.args, context));
		AvailabilityChangeEvent.publish(context, LivenessState.CORRECT);
	}
  ...
}

还有注意一点再ApplicationContext未创建完成前，指定的广播器initialMulticaster是无法替换的，默认是没有线程池的，只能串行执行。
而ApplicationContext创建成功后，你可以手动创建一个SimpleApplicationEventMulticaster来配置线程池，从而异步执行后续事件。

@Configuration
public class AsynchronousSpringEventsConfig {

   @Bean(name = AbstractApplicationContext.APPLICATION_EVENT_MULTICASTER_BEAN_NAME)
   public SimpleApplicationEventMulticaster simpleApplicationEventMulticaster() {
      SimpleApplicationEventMulticaster eventMulticaster = new SimpleApplicationEventMulticaster();
      eventMulticaster.setTaskExecutor(new SimpleAsyncTaskExecutor());
      return eventMulticaster;
   }

}

EventPublishingRunListener 事件

名称 触发方法 说明
ApplicationStartingEvent starting 初始化SpringApplication后，准备Environment前
ApplicationEnvironmentPreparedEvent environmentPrepared 准备Environment后，创建上下文前
ApplicationContextInitializedEvent contextPrepared 创建上下文并设置执行完ApplicationContextInitializer后，上下文loadSource前
ApplicationPreparedEvent contextLoaded 上下文loadSource后，上下文refresh前
ApplicationStartedEvent started 上下文refresh后，回调callRunners前
ApplicationReadyEvent running 上下文完全启动
ApplicationFailedEvent failed SpringApplication启动失败会被catch，再触发此事件

发布 ApplicationStartingEvent

public void multicastEvent(ApplicationEvent event) {
   multicastEvent(event, resolveDefaultEventType(event));
}

public void multicastEvent(final ApplicationEvent event, @Nullable ResolvableType eventType) {
  ResolvableType type = (eventType != null ? eventType : resolveDefaultEventType(event));
  Executor executor = getTaskExecutor();
  for (ApplicationListener<?> listener : getApplicationListeners(event, type)) {
    // 没有executor
    if (executor != null) {
      executor.execute(() -> invokeListener(listener, event));
    }
    else {
      invokeListener(listener, event);
    }
  }
}

protected void invokeListener(ApplicationListener<?> listener, ApplicationEvent event) {
  ErrorHandler errorHandler = getErrorHandler();
  if (errorHandler != null) {
    try {
      doInvokeListener(listener, event);
    }
  }
  else {
    // 简单的调用 ApplicationListener#onApplicationEvent
    doInvokeListener(listener, event);
  }
}

prepareEnvironment

private ConfigurableEnvironment prepareEnvironment(SpringApplicationRunListeners listeners,
      ApplicationArguments applicationArguments) {
   // 根据实例化SpringApplication时推断出的环境创建对应的Environment
   ConfigurableEnvironment environment = getOrCreateEnvironment();
   // 配置环境
   configureEnvironment(environment, applicationArguments.getSourceArgs());
   // 封装到PropertySource对象中
   ConfigurationPropertySources.attach(environment);
   // 发布ApplicationEnvironmentPreparedEvent事件
   listeners.environmentPrepared(environment);
   // 绑定"spring.main"配置
   bindToSpringApplication(environment);
   if (!this.isCustomEnvironment) {
      environment = new EnvironmentConverter(getClassLoader()).convertEnvironmentIfNecessary(environment,
            deduceEnvironmentClass());
   }
   // 将configurationProperties 放在 PropertySources 的第一个
   ConfigurationPropertySources.attach(environment);
   return environment;
}

configureEnvironment

protected void configureEnvironment(ConfigurableEnvironment environment, String[] args) {
   if (this.addConversionService) {
     // 创建并配置转换类 ApplicationConversionService
      ConversionService conversionService = ApplicationConversionService.getSharedInstance();
      environment.setConversionService((ConfigurableConversionService) conversionService);
   }
   // 配置PropertySource
   configurePropertySources(environment, args);
   // 读取profile
   configureProfiles(environment, args);
}

// 标准的Double-Check
public static ConversionService getSharedInstance() {
  ApplicationConversionService sharedInstance = ApplicationConversionService.sharedInstance;
  if (sharedInstance == null) {
    synchronized (ApplicationConversionService.class) {
      sharedInstance = ApplicationConversionService.sharedInstance;
      if (sharedInstance == null) {
        sharedInstance = new ApplicationConversionService();
        ApplicationConversionService.sharedInstance = sharedInstance;
      }
    }
  }
  return sharedInstance;
}

protected ConfigurableApplicationContext createApplicationContext() {
   Class<?> contextClass = this.applicationContextClass;
   if (contextClass == null) {
      try {
         switch (this.webApplicationType) {
         case SERVLET:

         case REACTIVE:

prepareContext

private void prepareContext(ConfigurableApplicationContext context, ConfigurableEnvironment environment,
      SpringApplicationRunListeners listeners, ApplicationArguments applicationArguments, Banner printedBanner) {
   // ConfigurableApplicationContext设置Environment
   context.setEnvironment(environment);
   // 设置beanNameGenerator、resourceLoader和classLoader、ApplicationConversionService
   postProcessApplicationContext(context);
   // 实例化SpringApplication 时的 ApplicationContextInitializers
   applyInitializers(context);
   // 发布ApplicationContextInitializedEvent 事件, 但是没有实际处理的listener
   listeners.contextPrepared(context);
   ConfigurableListableBeanFactory beanFactory = context.getBeanFactory();
   // 注册单例DefaultApplicationArguments
   beanFactory.registerSingleton("springApplicationArguments", applicationArguments);
   if (printedBanner != null) {
      // 注册单例Banner
      beanFactory.registerSingleton("springBootBanner", printedBanner);
   }
   if (beanFactory instanceof DefaultListableBeanFactory) {
      // 默认是无法重复定义
      // 可以手动设置"spring.main.allow-bean-definition-overriding=true"
      // 会在 prepareEnvironMent 中 bindToSpringApplication 修改
      ((DefaultListableBeanFactory) beanFactory)
            .setAllowBeanDefinitionOverriding(this.allowBeanDefinitionOverriding);
   }
   // 如果设置，只有显示设置 @Lazy(false) 才不会延迟初始化
   if (this.lazyInitialization) {
      context.addBeanFactoryPostProcessor(new LazyInitializationBeanFactoryPostProcessor());
   }
   // 默认是primarySources，即启动类
   Set<Object> sources = getAllSources();
   load(context, sources.toArray(new Object[0]));
   listeners.contextLoaded(context);
}

protected void load(ApplicationContext context, Object[] sources) {
  // 内有 AnnotatedBeanDefinitionReader 和 ClassPathBeanDefinitionScanner
  BeanDefinitionLoader loader = createBeanDefinitionLoader(getBeanDefinitionRegistry(context), sources);
  // 委托给 AnnotatedBeanDefinitionReader 注册
  loader.load();
}

private <T> void doRegisterBean(Class<T> beanClass, @Nullable String name,
			@Nullable Class<? extends Annotation>[] qualifiers, @Nullable Supplier<T> supplier,
			@Nullable BeanDefinitionCustomizer[] customizers) {
    // AnnotatedBeanDefinitionReader
		AnnotatedGenericBeanDefinition abd = new AnnotatedGenericBeanDefinition(beanClass);
		abd.setInstanceSupplier(supplier);
    // 解析 scope，通常为singleton。
		ScopeMetadata scopeMetadata = this.scopeMetadataResolver.resolveScopeMetadata(abd);
		abd.setScope(scopeMetadata.getScopeName());
    // AnnotationBeanNameGenerator 命名生成器
		String beanName = (name != null ? name : this.beanNameGenerator.generateBeanName(abd, this.registry));
    // 解析@Lazy @Primary @DependsOn @Role @Description
		AnnotationConfigUtils.processCommonDefinitionAnnotations(abd);
		BeanDefinitionHolder definitionHolder = new BeanDefinitionHolder(abd, beanName);
		definitionHolder = AnnotationConfigUtils.applyScopedProxyMode(scopeMetadata, definitionHolder, this.registry);
    // BeanDefinitionRegistry 注册 PrimarySource 的BeanDefinitionHolder
		BeanDefinitionReaderUtils.registerBeanDefinition(definitionHolder, this.registry);
	}

public final void refresh() throws BeansException, IllegalStateException {
  try {
    super.refresh();
  }
  catch (RuntimeException ex) {
    WebServer webServer = this.webServer;
    if (webServer != null) {
      webServer.stop();
    }
    throw ex;
  }
}

// ReactiveWebServerApplicationContext
public final void refresh() throws BeansException, IllegalStateException {
  try {
    super.refresh();
  }
  catch (RuntimeException ex) {
    WebServerManager serverManager = this.serverManager;
    if (serverManager != null) {
      serverManager.getWebServer().stop();
    }
    throw ex;
  }
}

执行 Runners 回调

ApplicationRunner与CommandLineRunner只是接口参数的区别。

private void callRunners(ApplicationContext context, ApplicationArguments args) {
  // 添加ApplicationRunner、CommandLineRunner
  runners.addAll(context.getBeansOfType(ApplicationRunner.class).values());
  runners.addAll(context.getBeansOfType(CommandLineRunner.class).values());
  AnnotationAwareOrderComparator.sort(runners);
  for (Object runner : new LinkedHashSet<>(runners)) {
    if (runner instanceof ApplicationRunner) {
      callRunner((ApplicationRunner) runner, args);
    }
    if (runner instanceof CommandLineRunner) {
      callRunner((CommandLineRunner) runner, args);
    }
  }
}

    private static final CountDownLatch preinitializationComplete = new CountDownLatch(1);

@Override
public void onApplicationEvent(SpringApplicationEvent event) {
   if (!Boolean.getBoolean(IGNORE_BACKGROUNDPREINITIALIZER_PROPERTY_NAME)
         && event instanceof ApplicationEnvironmentPreparedEvent && multipleProcessors()
         && preinitializationStarted.compareAndSet(false, true)) {
      performPreinitialization();
   }
   if ((event instanceof ApplicationReadyEvent || event instanceof ApplicationFailedEvent)
         && preinitializationStarted.get()) {
      try {
         preinitializationComplete.await();
      }
      catch (InterruptedException ex) {
         Thread.currentThread().interrupt();
      }
   }
}

private void performPreinitialization() {
		try {
			Thread thread = new Thread(new Runnable() {

				@Override
				public void run() {
          // 这些是耗时的组件，先后台提前预热下
					runSafely(new ConversionServiceInitializer());
					runSafely(new ValidationInitializer());
					runSafely(new MessageConverterInitializer());
					runSafely(new JacksonInitializer());
					runSafely(new CharsetInitializer());
					preinitializationComplete.countDown();
				}

				public void runSafely(Runnable runnable) {
					try {
						runnable.run();
					}
					catch (Throwable ex) {
					}
				}

			}, "background-preinit");
			thread.start();
		}
		catch (Exception ex) {
			preinitializationComplete.countDown();
		}
}

启动失败

启动失败会发送ApplicationFailedEvent事件

  1. LoggingApplicationListener：执行loggingSystem.cleanUp()清理资源
  2. ClasspathLoggingApplicationListener：如果开启了debug级别，输出 “Application failed to start with classpath: ”
  3. BackgroundPreinitializer：失败后可能初始化还未结束，CountDownLatch#await等待后台执行结束
  4. ConditionEvaluationReportLoggingListener：错误日志输出

总结

  1. Manifest.MF 提供了整个工程启动的元数据，配置的Main-Class启动类创建类加载器加载指定目录下的类，最后通过反射Start-Class的 main 方法，间接启动服务。
  2. Spring Boot 大量通过查找 ClassPath 下是否有相应的类来进行环境的推断。
  3. SPI 本质就是读取约定路径下的配置文件，SPI机制是扩展的基石。
  4. ApplicationContextInitializer是一个应用上下文刷新前的一个扩展点，可以额添加一些环境变量或者注册组件。执行时机是SpringApplication#prepareContext。
  5. ApplicationListener 是广泛运用在 Spring Framework 中的事件驱动机制。
  6. Spring Boot 中新增了 SpringApplicationRunListener，SpringApplicationRunListener本质是转发ApplicationListener。
    该接口定义了 SpringApplication的生命周期，会在每个生命周期的节点发送对应的事件。
  7. SpringApplication主流程：
    初始化 SpringApplication --> ApplicationStartingEvent -->
准备环境 --> ApplicationEnvironmentPreparedEvent -->
创建 ApplicationConext 初始化 --> ApplicationContextInitializedEvent -->
loadSource --> ApplicationPreparedEvent -->
SpringApplication#refresh --> ApplicationStartedEvent -->
callRunners --> ApplicationReadyEvent
失败 --> ApplicationFailedEvent
  8. EventPublishingRunListener 中的事件广播器没有taskExecutor 只能串行执行，等待应用上下文创建完毕后，事件的发布归还给应用上下文，这时的广播器可以自定义taskExecutor异步处理事件。
